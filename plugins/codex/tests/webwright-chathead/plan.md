# Critical Points

- [ ] CP1: The chathead loads in the authenticated Unraid webGUI without
  replacing or modifying the stock page content.
- [ ] CP2: Opening the chathead establishes the same-origin WebSocket connection
  to Codex app-server in the Incus container.
- [ ] CP3: The ChatGPT device-login flow can be initiated from the chathead.
- [ ] CP4: After authentication, a prompt streams a response and the thread can
  be resumed.
- [ ] CP5: A write approval or MCP elicitation renders in the drawer and cannot
  proceed without an explicit one-shot approval.
- [ ] CP6: The chathead remains available after navigating to another Unraid
  page.
- [ ] CP7: The drawer uses Aurora Light tokens and actual Aurora conversation,
  prompt, reasoning, plan, terminal, tool-call, permission, and context
  components without violet styling.
- [ ] CP8: The connection state is a tooltip-backed dot only, and the
  icon-only permission shield is inside PromptInput.
